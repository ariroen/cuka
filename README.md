# 🫡 Контракт-61: Диспетчер

**Voice-First CRM** для автоматизации учета кандидатов на военную службу.

## 🎯 Назначение

Система для контроля перемещения кандидатов:
- Учет источников прихода (реклама, прямые обращения)
- Контроль покупки билетов и прибытия
- Трекинг прохождения медицины (ВВК)
- Отметка об убытии в учебные центры

## 🔥 Ключевая концепция: "Voice-First"

Минимум ручного ввода! Бот понимает человеческую речь через Groq AI:

```
Ты: "Запиши Алексея, 89001234567, пришел с рекламы, билет купили"
Бот: ✅ Создает карточку, ставит источник "Реклама", статус билета "Куплен"
```

## 🛠 Технический стек

| Компонент | Технология |
|-----------|------------|
| Язык | Python 3.12 |
| Бот | aiogram 3.x (асинхронный) |
| База данных | SQLAlchemy + SQLite |
| AI/Голос | Groq API (Whisper + Llama 3.3) |
| Контейнеризация | Docker + Docker Compose |

## 📋 Функционал

### 1. Интеллектуальный ввод через ИИ
- **Транскрибация**: Голосовые сообщения → текст (Whisper)
- **LLM-Аналитик**: Понимание намерений (Llama 3.3)
- **Авто-заполнение**: ИИ сам создает карточки и ставит статусы

### 2. Цепочка контроля (воронка)
```
Источник → Билет (Нужен/Куплен/Прибыл) → Медицина → Обучение
```

### 3. Матрица статусов
В списке кандидатов видно: `Иванов И.И. | 🎫✅ | 🏥⏳ | 🪖❌`

### 4. Фото-архив
Прикрепление скриншотов билетов прямо к карточке кандидата.

### 5. Напоминания
Голосовая команда: *"Напомни проверить Петрова завтра в 10 утра"*

## 🚀 Быстрый старт

### Вариант 1: Docker (рекомендуется)

```bash
# 1. Скопируйте .env.example в .env
cp .env.example .env

# 2. Заполните .env своими ключами
# BOT_TOKEN=ваш_токен_бота
# GROQ_API_KEY=ваш_ключ_groq

# 3. Запустите
docker-compose up -d --build
```

### Вариант 2: Локальный запуск

```bash
# 1. Установите зависимости
pip install -r requirements.txt

# 2. Скопируйте и заполните .env
cp .env.example .env

# 3. Запустите бота
python bot.py
```

## 📖 Команды бота

| Команда | Описание |
|---------|----------|
